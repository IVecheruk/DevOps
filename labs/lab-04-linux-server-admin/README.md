## Цель работы

Изучить основы администрирования Linux-сервера и освоить практические приёмы работы с учётными записями пользователей, `sudo`, переменными окружения, `screen`, `systemd` и журналами системы.

## Теоретические сведения

В серверной среде Linux особенно важны навыки управления пользователями, разграничения прав доступа и контроля фоновых процессов. Утилиты `sudo` и `su` позволяют выполнять команды с повышенными привилегиями, а `screen` помогает поддерживать длительные процессы после отключения от терминала.

Система инициализации `systemd` отвечает за запуск и сопровождение сервисов, а также поддерживает таймеры, которые могут выполнять задачи по расписанию. Для анализа работы сервисов используется `journalctl`, а для проверки параметров расписания удобно применять `systemd-analyze calendar`.

### 2. Сравнение `sudo` и `su`, настройка ограниченных привилегий

Далее были проверены способы получения прав суперпользователя. Для пользователя `jarservice` было настроено правило, разрешающее выполнять `/usr/bin/id` от имени `root` без запроса пароля.

```bash
sudo id
sudo -i
su -c "id"
sudo visudo -f /etc/sudoers.d/jarservice-id
sudo chmod 440 /etc/sudoers.d/jarservice-id
sudo visudo -c
```

Фрагмент файла `sudoers`:

```text
jarservice ALL=(root) NOPASSWD: /usr/bin/id
```

### 4. Длительные процессы в `screen`

### 5. Создание `systemd` unit-файлов и просмотр журналов

На завершающем этапе были подготовлены файлы `monitoring.service` и `monitoring.timer`, после чего выполнены перечитывание конфигурации `systemd`, проверка расписания и просмотр журналов сервиса.

```ini
[Unit]
Description=Demo monitoring service
After=network.target

[Service]
Type=oneshot
ExecStart=/usr/bin/free -h

[Install]
WantedBy=multi-user.target
```

```ini
[Unit]
Description=Run monitoring.service on schedule

[Timer]
OnCalendar=*-*-* *:00:00
Persistent=true
Unit=monitoring.service

[Install]
WantedBy=timers.target
```

```bash
ps -p 1
sudo systemctl daemon-reload
systemd-analyze calendar "*-*-* *:00:00"
journalctl -u monitoring.service -n 20
journalctl -u monitoring.service --since "2026-01-01 00:00:00" --until "2026-12-31 23:59:59"
```
